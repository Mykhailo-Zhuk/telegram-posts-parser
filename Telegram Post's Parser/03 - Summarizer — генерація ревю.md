# 03 — Summarizer — генерація ревю

## Призначення

`summarize.py` читає `posts.json`, надсилає текст кожного поста до Claude API і генерує коротке 2–3 речення резюме. Результат записується у поле `review` та зберігається у `posts_reviewed.json`.

## Запуск

```bash
python summarize.py
```

Скрипт:
- Читає `posts.json`
- Для кожного поста без `review` робить запит до Claude API
- Записує результат у `review` поле
- Зберігає проміжний результат після **кожного** поста
- Якщо перервати — повтор продовжить з місця зупинки

## Системний промпт

```
Ти — редактор, який робить короткі огляди постів з Telegram-каналу.
Твоє завдання: написати 2–3 речення українською мовою, які передають суть поста.
Будь конкретним і інформативним. Уникай загальних фраз типу
"автор розповідає", "у цьому пості" або "це важливо".
Відповідай ТІЛЬКИ текстом ревю, без будь-яких пояснень, форматування чи маркдауну.
```

## Параметри Claude API

| Параметр | Значення | Опис |
|---|---|---|
| Модель | `claude-sonnet-4-20250514` | Оптимальний баланс якості та ціни |
| `max_tokens` | 300 | Достатньо для 2–3 речень |
| Обрізка тексту | 3000 символів | Економія токенів, головне — суть |
| Затримка між запитами | 0.5 сек | Униканням rate limits |
| Timeout | 30 сек | Максимум чекати на відповідь API |

## Логіка роботи

```
1. Завантажити posts.json
2. Для кожного поста:
   a. Перевірити наявність review
   b. Якщо review === null:
      i. Обрізати текст до 3000 символів
      ii. Надіслати запит до Claude API
      iii. Записати відповідь у post["review"]
      iv. Одразу зберегти posts_reviewed.json (проміжне збереження!)
      v. Зачекати 0.5 сек
   c. Якщо review уже існує — пропустити
3. Готово! posts_reviewed.json обновлений
```

> [!tip] Проміжне збереження
> Файл зберігається після **кожного** поста. Якщо скрипт перервати (Ctrl+C) — прогрес не втрачається. Повторний запуск продовжить з місця зупинки.

## Крайні випадки

| Ситуація | Поведінка |
|---|---|
| Пост без тексту | Ревю: "Пост без текстового вмісту." |
| Дуже короткий текст (< 10 символів) | Ревю: "Не вдалося згенерувати осмислене ревю." |
| Rate limit від Claude API (429) | Пауза 60 сек, повторна спроба |
| Помилка мережі | Пауза 10 сек, повторна спроба (до 3 разів) |
| Будь-яка інша помилка | Ревю: "Помилка генерації ревю: {error}" |
| Пост вже має review | Пропускається, economia токенів |

## Вартість запитів

Орієнтовно на 50 постів (середній текст ~500 символів):

| Показник | Значення |
|---|---|
| Середній вхід | ~400 токенів на пост |
| Середній вихід | ~60 токенів на пост |
| Всього (50 постів) | ~23,000 токенів |
| Вартість (claude-sonnet-4) | ~$0.07–0.10 |

> [!note] Оптимізація
> При великій кількості постів можна експериментувати:
> - `claude-haiku-4-5` — 10x дешевший, але якість нижча
> - Збільшити батч-розмір для паралельних запитів (але обережно з rate limits)
> - Пропускати занадто короткі пости

## Формат posts_reviewed.json

Те саме що `posts.json`, але з заповненим `review`:

```json
[
  {
    "id": 1042,
    "channel": "Tech UA",
    "text": "Повний текст...",
    "date": "2026-04-10T09:14:00+00:00",
    "link": "https://t.me/tech_ua/1042",
    "photo": "photos/1042.jpg",
    "review": "У Telegram запущена нова функція шифрування. Зміцнює безпеку чатів. Очікується глобальний розкат.",
    "read": false
  }
]
```

## Вкладання в FastAPI

Backend ендпоінт для запуску summarize через API:

```python
@app.post("/api/summarize")
async def summarize_posts():
    """Генерує ревю для всіх постів без review"""
    # Запускає тяжеліснеліський процес
    # Опціонально: запустити в background task
    subprocess.run(["python", "summarize.py"], check=True)
    return {"success": True, "message": "Summarization started"}
```
