# 02 — Fetcher — завантаження постів

## Призначення

`fetcher.py` підключається до Telegram через Telethon, читає пості з каналів/груп (налаштовані у `config.json`) та зберігає їх у `posts.json`.

Основна логіка:
- Читає `config.json` для отримання переліку каналів
- Читає `last_seen.json` для отримання ID останнього завантаженого поста
- Завантажує **тільки нові пості** (більші за last_seen ID)
- Обиває фото до `photos/` директорії
- Зберігає метадані у `posts.json`

## Запуск

### Тільки нові пості (після останнього запуску)

```bash
python fetcher.py
```

Логіка:
1. Читає останній ID з `last_seen.json`
2. Завантажує пості після цього ID
3. Оновлює `last_seen.json` до нового максимального ID
4. Результат у `posts.json`

### Всі пості з каналу (ігнорує last_seen)

```bash
python fetcher.py --all
```

Корисно при першому запуску або для ініціалізації нового каналу.

### Конкретний канал за індексом

```bash
python fetcher.py --channel 1
```

Завантажує тільки канал з індексом `1` з `config.json`. Корисно якщо потрібна швидка оновлення одного каналу.

Комбінація:
```bash
python fetcher.py --channel 1 --all
```

### Завантажити пості від конкретної дати

```bash
# Завантажити пості починаючи з 10 квітня 2026
python fetcher.py --from-date 2026-04-10

# Альтернативні формати дати:
python fetcher.py --from-date 10.04.2026       # DD.MM.YYYY
python fetcher.py --from-date 10/04/2026       # DD/MM/YYYY
python fetcher.py --from-date 10-04-2026       # DD-MM-YYYY
```

**Особливості**:
- Завантажує більше постів для фільтрації (до 500)
- Пропускає пости старіші за вказану дату
- **НЕ оновлює** `last_seen.json` при фільтрації за датою
- Корисно для заповнення історії з певного моменту

### Обмежити кількість постів

```bash
# Завантажити максимум 20 постів
python fetcher.py --limit 20

# Комбіновано: пості від дати з лімітом
python fetcher.py --from-date 2026-04-10 --limit 50
python fetcher.py --channel 1 --from-date 2026-04-01 --limit 100
```

**За замовчуванням** — лімід з `config.json` (зазвичай 50).

### Показати список каналів

```bash
python fetcher.py --list
```

Виведе:
```
📋 Канали у config.json:
  [0] Tech UA — tech_ua ◀ active
  [1] Стартапи UA — startups_ukraine
  [2] Приватна група — -1001234567890
```

## Комбінації команд

| Команда | Опис |
|---|---|
| `python fetcher.py` | Нові пості з активного каналу |
| `python fetcher.py --all` | Всі пості з активного каналу |
| `python fetcher.py --channel 1` | Нові пості з каналу #1 |
| `python fetcher.py --channel 1 --all` | Всі пості з каналу #1 |
| `python fetcher.py --from-date 2026-04-10` | Пості з активного каналу від 10 квітня |
| `python fetcher.py --from-date 2026-04-10 --limit 30` | Від дати з ліміту 30 |
| `python fetcher.py --channel 2 --from-date 10.04.2026 --limit 50` | Каналу #2, від дати, макс 50 |
| `python fetcher.py --list` | Показати всі канали |

## Що завантажується

Для кожного поста зберігається у `posts.json`:

| Поле | Тип | Опис |
|---|---|---|
| `id` | int | ID повідомлення в Telegram |
| `channel` | string | Назва каналу з `config.json` |
| `text` | string | Повний текст поста |
| `date` | string | Дата у форматі ISO 8601 (UTC) |
| `link` | string | Пряме посилання на пост у Telegram |
| `photo` | string / null | Шлях до завантаженого медіа (фото/відео/документ) або `null` |
| `review` | string / null | `null` — заповнюється `summarize.py` |
| `read` | boolean | `false` — керується через UI |

### Приклад запису

```json
{
  "id": 1042,
  "channel": "Tech UA",
  "text": "Повний текст поста з кількома реченнями...",
  "date": "2026-04-10T09:14:00+00:00",
  "link": "https://t.me/tech_ua/1042",
  "photo": "photos/1042.jpg",
  "review": null,
  "read": false
}
```

## Логіка роботи

```
1. Завантажити config.json → отримати список каналів
2. Завантажити last_seen.json → для кожного каналу отримати min_id
3. Визначити режим завантаження:
   a. Якщо --from-date → завантажувати від дати
   b. Якщо --all → завантажувати всі
   c. Інакше → тільки нові (з last_seen)
4. Підключитись до Telegram (Telethon, використовуючи session.session)
5. Для каналу:
   a. iter_messages(channel, min_id, limit=fetch_limit)
   b. Для кожного повідомлення:
      - Фільтрувати за датою якщо вказано --from-date
      - Обмежити кількість якщо вказано --limit
      - Сформувати post_link
      - Завантажити медіа → photos/{id}.jpg/mp4
      - Додати до списку
   c. Змерджити з існуючим posts.json (без дублів)
6. Оновити last_seen.json → але НЕ при фільтрації за датою
```

## Формат posts.json

```json
[
  {
    "id": 1042,
    "channel": "Tech UA",
    "text": "...",
    "date": "2026-04-10T09:14:00+00:00",
    "link": "https://t.me/tech_ua/1042",
    "photo": "photos/1042.jpg",
    "review": null,
    "read": false
  },
  {
    "id": 1041,
    "channel": "Tech UA",
    "text": "...",
    "date": "2026-04-09T14:22:00+00:00",
    "link": "https://t.me/tech_ua/1041",
    "photo": null,
    "review": null,
    "read": false
  }
]
```

## Типи медіа

Скрипт підтримує завантаження:

| Тип | Розширення | Приклад |
|---|---|---|
| Фото (JPEG) | `.jpg` | `photos/1042.jpg` |
| Відео | `.mp4` | `photos/1042.mp4` |
| Документи | оригінальне | `photos/1042.pdf`, `photos/1042.zip` |

## Крайні випадки

| Ситуація | Поведінка |
|---|---|
| Пост без тексту і медіа | Пропускається |
| Пост тільки з відео / аудіо | `photo: null`, текст збережений |
| Пост з кількома фото | Завантажується тільки перше |
| Приватний канал без username | Посилання у форматі `t.me/c/{channel_id}/{msg_id}` |
| Перший запуск (немає last_seen) | Завантажує останні `limit` постів |
| Повторний запуск без нових постів | Виводить повідомлення "No new posts", файл не змінюється |
| Канал більше недоступний | Помилка Telethon, скрипт пропускає канал |
| Limit rate від Telegram | Автоматична пауза, повторна спроба |
| Фільтрація за датою без результатів | Повертає пустий список |

## Параметри config.json

```json
{
  "limit": 50
}
```

- `limit` — максимум постів за замовчуванням, якщо не вказано `--limit`
- Для великих обсягів збільш `limit`, але пам'ятай про rate limits Telegram
- Рекомендовано: 25–100
