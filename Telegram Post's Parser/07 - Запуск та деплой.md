# 07 — Запуск та деплой

## Локальний запуск (розробка)

### Крок 1: Встановлення залежностей

**Frontend**:
```bash
npm install
```

**Backend**:
```bash
pip install -r backend/requirements.txt
# або
pip install fastapi uvicorn telethon anthropic pydantic
```

### Крок 2: Налаштування `.env`

Створи файл `.env` у корені проекту (скопіюй `.env.example`):

```bash
TG_API_ID=12345678
TG_API_HASH=abcdef1234567890abcdef1234567890
ANTHROPIC_API_KEY=sk-ant-...
```

### Крок 3: Першій запуск — завантажити пості

```bash
# Терміналу 1 — Backend API
python -m uvicorn backend.main:app --reload --port 8001

# Терміналу 2 — Завантажити пості
python fetcher.py

# Терміналу 2 (після fetcher завершить) — Генерувати ревю
python summarize.py

# Терміналу 3 — Frontend (Vite dev server)
npm run dev
# → відкри http://localhost:5173
```

**Інтерфейс буде доступний**: http://localhost:5173

> **Примітка**: Vite автоматично проксирує `/api/*` запити на бекенд (див. `vite.config.ts`).

## Запуск CLI скриптів

### fetcher.py

```bash
# Тільки нові пості (після останнього запуску, з last_seen)
python fetcher.py

# Всі пості з каналу (ігнорує last_seen)
python fetcher.py --all

# Конкретний канал за індексом
python fetcher.py --channel 1

# Показати список всіх налаштованих каналів
python fetcher.py --list
```

### summarize.py

```bash
# Генерувати ревю для всіх постів без review
python summarize.py
```

Скрипт:
- Пропускає пості, які вже мають `review` — безпечно перезапускати
- Зберігає проміжний результат після кожного поста
- Якщо перерватись — повтор продовжить з місця зупинки

### Рекомендована схема деплою

#### Варіант A — Vercel + Railway (РЕКОМЕНДОВАНИЙ)

**Плюси:**
- ✅ Безкоштовний tier для обох сервісів
- ✅ Автоматичний деплой з GitHub
- ✅ Сертифікати SSL за замовчуванням
- ✅ Simple env vars management

**Крок 1: Підготовка GitHub**

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/telegram-posts-parser.git
git push -u origin main
```

**Крок 2: Деплой Backend на Railway**

1. Зайди на https://railway.app
2. Натисни **"Start New Project"** → **"Deploy from GitHub"**
3. Підключи GitHub та виберу репо
4. Railway автоматично детектує `Procfile` та `requirements.txt`

**Встановлення Environment Variables у Railway dashboard:**

```
TG_API_ID = 12345678
TG_API_HASH = abcdef...
ANTHROPIC_API_KEY = sk-ant-...
```

> **Важливо**: Telethon потребує `session.session` файл для avторизації. Перший запуск вимагає інтерактивного введення коду. Рішення:
> 1. Авторизуйся локально: `python fetcher.py` (введи код з Telegram)
> 2. Кодуй `session.session` в base64
> 3. Зберегти як Railway Secret: `SESSION_BASE64`
> 4. Backend декодує при старті

**Крок 3: Деплой Frontend на Vercel**

1. Зайди на https://vercel.com
2. Натисни **"New Project"**
3. Імпортуй GitHub репо
4. Vercel автоматично детектує Vite проект

**Build Settings (мають бути автоматичні)**:
```
Framework: Vite
Build Command: npm run build
Output Directory: dist
```

**Environment Variables у Vercel**:
```
VITE_API_URL = https://your-railway-app.railway.app/api
```

Натисни **Deploy** → готово! 🚀

**Результат:**
- Frontend: `https://your-app.vercel.app`
- Backend API: `https://your-app.railway.app`

#### Варіант B — Heroku (більш не free, але простий)

Heroku більше не має free tier, але для приватного використання можна:

```bash
heroku create your-app-name
heroku config:set TG_API_ID=...
heroku config:set TG_API_HASH=...
heroku config:set ANTHROPIC_API_KEY=...

git push heroku main
heroku logs --tail
```

#### Варіант C — ручна VPS (максимальна контроль)

На своїй VPS (DigitalOcean, Hetzner, AWS EC2):

```bash
# На сервері
git clone https://github.com/YOUR_USERNAME/telegram-posts-parser.git
cd telegram-posts-parser

# Встанови Python + Node
apt-get install python3.12 python3.12-venv nodejs npm

# Backend
python3 -m venv venv
source venv/bin/activate
pip install -r backend/requirements.txt
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 &

# Frontend
npm install && npm run build
# Serve dist/ через nginx

# Cron для автоматичного оновлення (кожні 2 години)
0 */2 * * * cd /path/to/repo && python fetcher.py && python summarize.py >> /tmp/digest.log 2>&1
```

Потім налаштуй nginx як reverse proxy для API.

- [ ] Отримано `TG_API_ID`, `TG_API_HASH` з https://my.telegram.org

- [ ] Заповнено `.env` файл

- [ ] `summarize.py` завершив (появилися ревю у `posts_reviewed.json`)
- [ ] Backend запущен: `python -m uvicorn backend.main:app --reload`
- [ ] Frontend запущен: `npm run dev`
- [ ] UI відображає пости коректно

### Тестування API endpoints

```bash
# Отримати всі пості
curl http://localhost:8001/api/posts

# Позначити пост як прочитаний
curl -X PUT http://localhost:8001/api/posts/1042/read \
  -H "Content-Type: application/json" \
  -d '{"read": true}'

# Завантажити нові пості
curl -X POST "http://localhost:8001/api/fetch?channel=tech_ua&limit=50"

# Генерувати ревю
curl -X POST http://localhost:8001/api/summarize
```

## Автоматизація оновлень

### GitHub Actions (для Railway)

Автоматичний запуск `fetcher.py` та `summarize.py` кожні 6 годин:

```yaml
# .github/workflows/update-digest.yml
name: Update Digest
on:
  schedule:
    - cron: '0 */6 * * *'   # кожні 6 годин
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.12'

      - name: Install dependencies
        run: pip install telethon anthropic

      - name: Fetch & Summarize
        env:
          TG_API_ID: ${{ secrets.TG_API_ID }}
          TG_API_HASH: ${{ secrets.TG_API_HASH }}
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          python fetcher.py
          python summarize.py

      - name: Commit changes
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add posts_reviewed.json photos/
          git diff --staged --quiet || git commit -m "chore: update digest"
          git push
```

### Локальне автоматичне оновлення (cron)

На своїй машині чи VPS:

```bash
crontab -e
```

```cron
# Оновлення кожні 2 години
0 */2 * * * cd /path/to/telegram-posts-parser && /usr/bin/python3 fetcher.py && /usr/bin/python3 summarize.py >> logs/digest.log 2>&1
```
