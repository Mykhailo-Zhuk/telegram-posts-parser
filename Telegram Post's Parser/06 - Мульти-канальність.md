# 06 — Мульти-канальність

## Концепція

Проект підтримує **кілька Telegram-каналів і груп одночасно**. Управління каналами здійснюється через `config.json` (для CLI скриптів) та динамічно у UI (для користувачів).

## config.json — визначення каналів

```json
{
  "channels": [
    { "name": "Tech UA",        "id": "tech_ua" },
    { "name": "Стартапи UA",    "id": "startups_ukraine" },
    { "name": "Приватна група", "id": -1001234567890 },
    { "name": "Дружба",         "id": -1009876543210 }
  ],
  "active_channel": 0,
  "limit": 50
}
```

| Поле | Тип | Опис |
|---|---|---|
| `channels` | Array[Object] | Масив налаштованих каналів |
| `name` | string | Відображувана назва (у UI та логах) |
| `id` | string / int | Username без `@` або числовий ID (для приватних) |
| `active_channel` | int | Індекс каналу за замовчуванням для CLI |
| `limit` | int | Максимум постів за один запит (за замовчуванням) |

## Типи каналів

### Публічний канал

```json
{ "name": "Tech UA", "id": "tech_ua" }
```

- `id` — username без `@` (рядок)
- Посилання на пост: `https://t.me/tech_ua/1042`
- Доступний для всіх у Telegram

### Приватна група / канал

```json
{ "name": "Моя приватна gruppe", "id": -1001234567890 }
```

- `id` — числовий ID (від'ємне число)
- Посилання на пост: `https://t.me/c/1234567890/42`
- Потрібна авторизація Telethon
- Отримати ID: `python fetcher.py --list`

## CLI — управління каналами

### Показати всі канали

```bash
python fetcher.py --list
```

Виведе:
```
📋 Канали у config.json:
  [0] Tech UA — tech_ua ◀ active
  [1] Стартапи UA — startups_ukraine
  [2] Приватна група — -1001234567890
  [3] Дружба — -1009876543210
```

### Завантажити пості з конкретного каналу

```bash
# Тільки нові пості з каналу 1
python fetcher.py --channel 1

# Всі пості з каналу 2 (ігнорує last_seen)
python fetcher.py --channel 2 --all

# Тільки нові пості з активного каналу (з config.json)
python fetcher.py

# Всі пості зі всіх каналів (потрібно запустити для кожного)
python fetcher.py --all
```

### Завантажити пості від конкретної дати

```bash
# Пості з каналу 1 від 10 квітня
python fetcher.py --channel 1 --from-date 2026-04-10

# Альтернативні формати дати:
python fetcher.py --channel 0 --from-date 10.04.2026       # DD.MM.YYYY
python fetcher.py --channel 2 --from-date 10/04/2026       # DD/MM/YYYY

# З ліміту постів
python fetcher.py --channel 1 --from-date 2026-04-01 --limit 100
```

**Особливості фільтрації за датою:**
- ✅ Завантажує більше постів для точної фільтрації
- ✅ Підтримує кілька форматів: `2026-04-10`, `10.04.2026`, `10/04/2026`
- ✅ **НЕ оновлює** `last_seen.json` (для лнакопичення історії)

### Обмежити кількість постів

```bash
# Завантажити макс 20 постів з каналу 1
python fetcher.py --channel 1 --limit 20

# За замовчуванням — значення з config.json (зазвичай 50)
```

## Комбінації команд з параметрами

| Команда | Опис |
|---|---|
| `python fetcher.py` | Нові пості з активного каналу |
| `python fetcher.py --all` | Всі пості з активного каналу |
| `python fetcher.py --list` | Показати всі канали |
| `python fetcher.py --channel 1` | Нові пості з каналу #1 |
| `python fetcher.py --channel 1 --all` | Всі пості з каналу #1 |
| `python fetcher.py --channel 1 --limit 20` | Макс 20 нових постів з каналу #1 |
| `python fetcher.py --from-date 2026-04-10` | Пості з активного каналу від 10 квітня |
| `python fetcher.py --channel 2 --from-date 10.04.2026` | Каналу #2 від дати |
| `python fetcher.py --channel 1 --from-date 2026-04-10 --limit 50` | Каналу #1, від дати, макс 50 |

## UI — dinamicsс перемикання каналів

Канали генеруються **динамічно** з даних у `posts_reviewed.json`:

```
┌─────────────────────────────────────────────────┐
│  Header                                         │
│  ┌───────┬────────┬──────────┬──────────────┐   │
│  │ Всі   │ Tech   │ Стартапи │ Приватна    │   │
│  │ (123) │ (45)   │ (62)     │ (16)        │   │
│  └───────┴────────┴──────────┴──────────────┘   │
│                                                 │
│  У скобках — кількість непрочитаних постів    │
└─────────────────────────────────────────────────┘
```

Логіка в React (`App.tsx`):

1. Завантажити `posts_reviewed.json` з API
2. Екстрактити унікальні значення `channel` з постів
3. Сформувати таблетки для кожного каналу
4. Додати таблетку "Всі канали" на початок
5. При кліку на таблетку — оновити фільтр `activeChannel`

**Переваги динамічного підходу:**
- ✅ Не потрібно перезапускати UI при додаванні нового каналу
- ✅ Автоматичне видалення каналів, які більше не мають постів
- ✅ Лічильники непрочитаних оновлюються в реальному часі

## last_seen.json при кількох каналах

Маркери зберігаються окремо для кожного каналу за ключем `str(channel_id)`:

```json
{
  "tech_ua": 1087,
  "startups_ukraine": 445,
  "-1001234567890": 89,
  "-1009876543210": 234
}
```

**Логіка оновлення:**
- `python fetcher.py --channel 1` оновлює **тільки** маркер для цього каналу
- `python fetcher.py --from-date ...` **НЕ оновлює** маркер (для накопичення історії)
- `python fetcher.py --all` ігнорує маркер взагалі

```json
{
  "tech_ua": 1087,
  "startups_ukraine": 501,    ← Оновлено після fetcher.py --channel 1!
  "-1001234567890": 89,
  "-1009876543210": 234
}
```

## posts.json при кількох каналах

Всі пості з усіх каналів зберігаються у **одному** файлі:

```json
[
  { "id": 1042, "channel": "Tech UA", "text": "...", ... },
  { "id": 445,  "channel": "Стартапи UA", "text": "...", ... },
  { "id": 89,   "channel": "Приватна група", "text": "...", ... },
  { "id": 1041, "channel": "Tech UA", "text": "...", ... },
  ...
]
```

Файл постійно розраститься. Опціонально можна додати логіку видалення старих постів:

```python
# Залишити останні 1000 постів (або по даті)
posts = sorted(posts, key=lambda p: p["date"], reverse=True)[:1000]
```

## Стратегія завантаження всіх каналів

### Варіант A — Shell скрипт (локально)

```bash
#!/bin/bash
# update_all.sh
set -e

echo "=== Updating all channels ==="

# Отримуємо кількість каналів з config.json
CHANNELS=$(python -c "import json; print(len(json.load(open('config.json'))['channels']))")

for i in $(seq 0 $((CHANNELS-1))); do
  echo "📡 Канал $i..."
  python fetcher.py --channel $i || true
done

echo "📝 Generating reviews..."
python summarize.py

echo "✅ All channels updated!"
```

Запуск:
```bash
chmod +x update_all.sh
./update_all.sh
```

### Варіант B — FastAPI endpoint (на сервері)

```python
@app.post("/api/update-all")
async def update_all():
    """Оновлює всі канали та генерує ревю"""
    try:
        subprocess.run(["python", "fetcher.py"], check=True, timeout=300)
        subprocess.run(["python", "summarize.py"], check=True, timeout=600)
        return {"success": True, "message": "All channels updated"}
    except subprocess.CalledProcessError as e:
        return {"success": False, "error": str(e)}, 500
```

Використання:
```bash
curl -X POST http://localhost:8001/api/update-all
```

## Одновіменне завантаження кількох каналів

Можна паралелізувати для швидкості (обережно з rate limits!):

```python
import concurrent.futures
import json

def fetch_channel(channel_id):
    subprocess.run(["python", "fetcher.py", "--channel", str(channel_id)], check=True)

with open("config.json") as f:
    config = json.load(f)
    num_channels = len(config["channels"])

with concurrent.futures.ThreadPoolExecutor(max_workers=2) as executor:
    futures = [executor.submit(fetch_channel, i) for i in range(num_channels)]
    for future in concurrent.futures.as_completed(futures):
        future.result()
```

## Практичні приклади

### Заповнити історію з певної дати

```bash
# Завантажити все з каналу 0 з березня 2026
python fetcher.py --channel 0 --from-date 2026-03-01
python summarize.py
```

### Швидке оновлення одного каналу

```bash
# Отримати тільки останні 10 постів з каналу 1
python fetcher.py --channel 1 --limit 10
python summarize.py
```

### Накопичити пости з поточного місяця

```bash
# Оновити всі канали за поточний місяць (з 1 числа до сьогодні)
for i in 0 1 2; do
  python fetcher.py --channel $i --from-date 2026-04-01
done
python summarize.py
```

## Видалення постарілих постів

Опціонально для економії місця:

```python
from datetime import datetime, timedelta

now = datetime.now(timezone.utc)
cutoff = now - timedelta(days=30)

posts = [p for p in posts if datetime.fromisoformat(p["date"]) > cutoff]
```

Або залишити останні N постів:
```python
posts = posts[-5000:]  # останні 5000
```
