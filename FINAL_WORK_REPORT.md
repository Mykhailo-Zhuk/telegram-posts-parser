## 🔧 **ТЕХНІЧНІ ДЕТАЛІ:**

### **Архітектура системи:**
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Scheduler     │───▶│   Pipeline      │───▶│   Telegram      │
│   (кожні 2 год) │    │   (обробка)     │    │   (fetcher)     │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   Cleanup       │◀───│   AI Summarize  │◀───│   Data Storage  │
│   (очищення)    │    │   (Claude API)  │    │   (JSON files)  │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

### **Розклад роботи:**
- **Кожні 2 години:** Отримання та обробка нових постів
- **Щодня о 3:00:** Очищення тимчасових файлів
- **Щонеділі о 4:00:** Резервне копіювання даних
- **Постійно:** Моніторинг здоров'я системи

### **Політика очищення:**
- **Тимчасові файли:** 1 день retention
- **Медіа файли:** 7 днів, максимум 10 GB
- **JSON дані:** 30 днів, архівування перед видаленням
- **Логи:** 30 днів, ротація при 10 MB

## 🚀 **ІНСТРУКЦІЇ ДЛЯ ЗАПУСКУ:**

### **Простий запуск:**
```bash
cd telegram-posts-parser
python3 scheduler.py
```

### **Як сервіс (рекомендовано):**
```bash
# Створи systemd сервіс
sudo nano /etc/systemd/system/telegram-review.service

# Вміст:
[Unit]
Description=Telegram Review Scheduler
After=network.target

[Service]
Type=simple
User=openclaw
WorkingDirectory=/home/openclaw/telegram-posts-parser
ExecStart=/usr/bin/python3 scheduler.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target

# Запусти
sudo systemctl enable telegram-review
sudo systemctl start telegram-review
```

### **Моніторинг:**
```bash
# Перегляд логів
tail -f pipeline.log

# Статистика
cat pipeline_stats.json

python3 cleanup.py --dry-run
python3 cleanup.py --force
```
